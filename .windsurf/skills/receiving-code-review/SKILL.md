name: receiving-code-review
description: Responds to review feedback constructively; preserves the plan and spec intent.
---

# Process
- Acknowledge and categorize: correctness, clarity, style, nit.
- For correctness/design issues: update the plan/design doc and code.
- For style/nits: apply if consistent with repo standards.
- Batch commits logically; keep history clean.

# Verify
- Re-run typecheck/lint/tests after changes.
- Update PR notes and screenshots as needed.

# Exit criteria
- Reviewer concerns addressed; approvals obtained.
